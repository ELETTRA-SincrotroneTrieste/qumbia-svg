Qt SVG cumbia integration module
Write a cumbia-svg application

Create the drawing

You shall use an SVG editor to make the drawing. The svg file can then be parsed by the application. Compiling the svg file into a Qt resource is an option. In that case, the following steps are necessary

Create a new cumbia project

cumbia new project

Fill the form and create the project structure.

Add the resource to the .pro file

From the Qt creator, add a new file to the project, of type Qt Resource File. Then, add the svg source to the resource file. The qrc and the svg files should appear under the Resources tree in your project. Clicking on the svg file, the Qt creator should display the drawing.

Configure the project to use cumbia-svg

Open the Qt *.pro* project file. If the skeleton has been created with the cumbia new project wizard, a couple of include lines should be found at the top of the file:

include (/usr/local/cumbia-libs/include/qumbia-epics-controls/qumbia-epics-controls.pri)
include (/usr/local/cumbia-libs/include/qumbia-tango-controls/qumbia-tango-controls.pri)

In this example, support for Tango and EPICS control systems is included.

Make sure that the QT lines contains the xml and opengl support:

QT += xml opengl

As a final step, in order to use the cumbia svg module, we will rely on pkgconfig to locate and load the qumbia-svg dependency:

CONFIG+=link_pkgconfig
packagesExist(qumbia-svg) {
PKGCONFIG += qumbia-svg
message("qusvgexample: found module qumbia-svg ")
} else {
error("qusvgexample: module qumbia-svg is required - check PKG_CONFIG_PATH")
}

Write the code

The tutorial describes an example. The class name is QuSvgSynoptic. We will write the code to load the svg into a QWidget.

Ingredients

  • QuSvgView a QGraphicsView with zooming and scene events management;
  • QuSvgItemEventHandler to process left item clicks and contextual menu events from the QuGraphicsSvgItem items in the scene.
  • QuSvg, the module main class combining all the components.

Header file

In our example, the main class implements the QuSvgDataListener interface, so that new data from the engines is received and processed therein.

More QuSvgDataListener implementations can be provided and registered with QuSvg with the QuSvg::addDataListener(const QString &id, QuSvgDataListener <em>l) method. Those listeners are associated to a specific *id of a svg node and will be invoked in order to process data changes concerning that id.

In this example, let QuSvgSynopticGlobalListener be a QuSvgDataListener in charge of receiving all data updates that are not automatically managed by the library itself. In effect, simple data changes, such as a value changing a label text or an item fill or stroke color change, can automatically update the associated graphics item, as you have learnt from the QuSvg class documentation.

class QuSvgSynopticGlobalListener : public QuSvgDataListener {
public:
// QuSvgDataListener interface
bool onUpdate(const QuSvgResultData &res, QuDom *qudom);
void onError(const QString &msg);
};

The listener will be registered later through the call to QuSvg::addDataListener

The header file of the main class named QuSvgSynoptic shall resemble this:

#include <qulogimpl.h>
#include <cucontrolsfactorypool.h>
#include <cumbiatango.h>
#include <cumbiaepics.h>
#include <qusvgdatalistener.h>
class CuData;
class CumbiaPool;
class QuSvg;
class QuSvgSynoptic : public QWidget, public QuSvgDataListener
{
Q_OBJECT
public:
explicit QuSvgSynoptic(CumbiaPool *cu_p, QWidget *parent = 0);
~QuSvgSynoptic();
// QuSvgDataListener interface
public:
bool onUpdate(const QuSvgResultData &res, QuDom *qudom);
void onError(const QString &msg);
private:
QuSvg *m_qu_svg;
CumbiaPool *cu_pool;
QuLogImpl m_log_impl;
CuControlsFactoryPool m_ctrl_factory_pool;

The cumbia-svg initialization code is placed in the class constructor in the cpp file:

// includes
#include <QGridLayout>
// cumbia includes...
#include <cumbiapool.h>
#include <cumbiaepics.h>
#include <cumbiatango.h>
#include <cuepcontrolsreader.h>
// more cumbia includes...
//
// cumbia svg includes
#include <qusvgview.h>
#include <qusvg.h>
#include <qusvglayerhelper.h>
#include <qusvgconnectionspool.h>
#include <qusvgitemeventhandler.h>
#include <qusvgwriteactionprovider.h>
#include <qusvghelperappactionprovider.h>
QuSvgSynoptic::QuSvgSynoptic(CumbiaPool *cumbia_pool, QWidget *parent) :
QWidget(parent)
{
// ... cumbia initialization
// ...
// svg components instantiation and connection
//
QGridLayout *lo = new QGridLayout(this);
QuSvgView *svgview = new QuSvgView(this);
m_qu_svg = new QuSvg(svgview);
// QuSvgView is the source of click and contextual menu events
QuSvgItemEventHandler *item_event_han = new QuSvgItemEventHandler(svgview);
item_event_han->addActionProvider(new QuSvgHelperAppActionProvider(this, m_qu_svg->quDom(), m_qu_svg->getConnectionsPool()));
item_event_han->addActionProvider(new QuSvgWriteActionProvider(this, m_qu_svg->quDom(), cu_pool, m_ctrl_factory_pool));
connect(item_event_han, SIGNAL(error(QString,QString)), this, SLOT(onItemEventHandlerError(QString, QString)));
m_qu_svg->init(cu_pool, m_ctrl_factory_pool);
lo->addWidget(svgview, 0,0, 5, 5 );
// load the svg file:
bool ok = m_qu_svg->loadFile(":src/drawing.svg");
if(ok) {
m_qu_svg->addDataListener(this);
m_qu_svg->addDataListener("mysvg", new QuSvgSynopticGlobalListener);
}
else {
// provide error message
}
